# 默认矩阵尺寸更新

## 更改内容

将基准测试的默认矩阵/批次尺寸从较小的值更新为更大、更实用的尺寸。

### 之前的默认值
```yaml
batch_sizes: [256, 512, 1024, 2048, 4096]

operators:
  elementwise_add:
    sizes: [256, 512, 1024, 2048, 4096]
  gemm:
    matrix_sizes: [256, 512, 1024, 2048, 4096]
```

### 更新后的默认值
```yaml
batch_sizes: [1024, 2048, 4096]

operators:
  elementwise_add:
    sizes: [1024, 2048, 4096]
  gemm:
    matrix_sizes: [1024, 2048, 4096]
```

## 更改原因

1. **更实际的工作负载**: 1024x1024的矩阵更接近实际应用场景
2. **更好的性能测试**: 较大的矩阵能更好地展现GPU的计算能力
3. **减少测试时间**: 去掉了较小的尺寸，专注于有意义的性能测试
4. **内存带宽测试**: 较大的矩阵能更好地测试内存带宽利用率

## 影响

- **GEMM基准测试**: 现在从1024x1024矩阵开始，而不是256x256
- **Element-wise操作**: 现在从1024个元素开始测试
- **测试时间**: 略有减少，因为测试的尺寸数量减少了
- **性能分析**: 更专注于实际应用场景的性能

## 自定义配置

如果需要测试其他尺寸，可以修改`config.yaml`文件：

```yaml
# 添加更多尺寸
operators:
  gemm:
    matrix_sizes: [512, 1024, 2048, 4096, 8192]

# 或者测试更小的尺寸
operators:
  gemm:
    matrix_sizes: [256, 512, 1024]
```

## 验证

运行基准测试确认新的默认尺寸：
```bash
make bench-gemm
# 输出: Matrix sizes: 1024, 2048, 4096

make bench-elementwise-add  
# 使用批次尺寸: 1024, 2048, 4096
```
